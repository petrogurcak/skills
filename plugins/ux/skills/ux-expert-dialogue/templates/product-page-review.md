### Section 1: Product Gallery

**User provides:**
```
Photo count: ____
Photo types: [ ] Product only  [ ] In-context  [ ] Detail/zoom  [ ] 360  [ ] Video  [ ] UGC
Zoom: [ ] Hover zoom  [ ] Click zoom  [ ] Pinch zoom (mobile)  [ ] No zoom

```

**Expert evaluates:**

**Checklist:**
- [ ] Minimum 3 photos (ideal 5-8 for physical products)
- [ ] Hero shot: product clearly visible, white/clean background
- [ ] Context shot: product in use/environment
- [ ] Detail shot: close-ups of texture, material, features
- [ ] Scale shot: product next to reference object (for size)
- [ ] Zoom functionality works (hover or click)
- [ ] Video present (product demo, 30-60 seconds)
- [ ] User-generated content included (real customer photos)
- [ ] Images load fast (<2 seconds)
- [ ] Gallery navigation obvious (thumbnails, arrows, dots)

**Challenge questions:**
1. "Kolik fotek mas? Minimum 3, ideal 5-8. Kazda dalsi fotka = +3% konverze"
2. "Mas kontextovou fotku (produkt v pouziti)? Stock na bilem pozadi nestaci"
3. "Muze zakaznik PRIBLIZIT detail? Baymard: 56% uzivatelu zkusi zoom"
4. "Kde je video? Product video zvysuje add-to-cart o 37%"

**Common mistakes:**
- Only 1-2 photos (insufficient for decision-making)
- Only white-background shots (no context)
- No zoom (56% of users attempt zoom - Baymard)
- Low resolution images (can't see detail)
- No video (missing 37% add-to-cart lift)
- Stock photos instead of real product
- Slow loading images (users leave after 3 seconds)
- No UGC (user-generated photos = +25% trust)

**Data to cite:**
- Baymard: 56% of users attempt zoom on product images
- Each additional product photo: +3% conversion (Shopify data)
- Product video: +37% add-to-cart rate (Wyzowl)
- UGC photos: +25% purchase confidence (TurnTo Networks)
- Context images (product in use): +22% conversion vs white-only (ConversionXL)
- 360-degree views: -27% return rate (Threekit)

**If insufficient photos:**
"Mas [N] fotek. Kazda dalsi = +3% konverze.

**Optimalni gallery (5-8 fotek):**
1. **Hero shot** - produkt, ciste pozadi, jasne co to je
2. **Context shot** - produkt v realnem pouziti (lifestyle)
3. **Detail shots (2-3)** - textura, material, klicove features
4. **Scale shot** - vedle referencniho predmetu
5. **Video** - 30-60s demo, unboxing, how-to-use
6. **UGC** - realne fotky od zakazniku

**Zoom:** Hover zoom (desktop), pinch zoom (mobile) je POVINNY.

**Expected:** +15-37% add-to-cart rate."

**Expected impact of fixing:** +15-37% add-to-cart

### Section 2: Title & Description

**User provides:**
```
Product title: "_________________"
Description length: ____ words
Description format: [ ] Paragraph  [ ] Bullets  [ ] Both  [ ] Tabs
Key benefits highlighted: [ ] Yes  [ ] No
SEO optimized: [ ] Yes  [ ] No
```

**Expert evaluates:**

**Checklist:**
- [ ] Title is benefit-focused (not just product name)
- [ ] Title includes key search terms (SEO)
- [ ] Short description (2-3 sentences) above fold
- [ ] Bullet points for key benefits (scannable)
- [ ] Benefits before features ("What you get" before "What it is")
- [ ] Sensory language used (feel, taste, experience)
- [ ] Technical specs in separate tab/section
- [ ] Size/dimensions clearly stated
- [ ] Material/ingredients listed

**Challenge questions:**
1. "Je titulek benefit-focused nebo jen nazev produktu?"
2. "Jsou nejdulezitejsi benefity videt BEZ scrollovani?"
3. "Pouzivas bullet points? 79% uzivatelu skenuje, necte"
4. "Jsou features prevlecene za benefity? 'Bavlna 100%' vs 'Pritulne mekka bavlna pro pohodli cely den'"

**Common mistakes:**
- Title is just product name (no benefit)
- Wall of text description (no one reads)
- Features only, no benefits ("400ml" vs "Enough for 2 servings")
- Technical jargon (user doesn't understand)
- Important info below fold
- No bullet points (79% scan, not read - Nielsen)
- Missing dimensions/specs (common return reason)

**Data to cite:**
- Nielsen: 79% of users scan, 16% read word-by-word
- Bullet points: +47% comprehension vs paragraphs (NNGroup)
- Benefit-focused descriptions: +23% add-to-cart (Shopify study)
- Sensory language: +18% purchase intent (Journal of Consumer Research)
- Above-fold key info: +12% engagement (Baymard)

**If wall of text:**
"Popis je wall of text. 79% uzivatelu SKENUJE, NECTE.

**Fix structure:**
1. **Short description (2-3 vety):** Nejdulezitejsi benefit nad foldem
2. **Bullet points (3-5):** Klicove benefity (ne features!)
3. **Tabs:** Detailni popis | Specifikace | Recenze

**Pravidlo benefit vs feature:**
- Feature: '100% bavlna, 300g'
- Benefit: 'Pritulne mekka bavlna pro pohodli cely den'

**Expected:** +23-47% engagement s popisem."

**Expected impact of fixing:** +12-47% engagement

### Section 3: Price & Availability

**User provides:**
```
Price display: $____
Compare-at price: [ ] Yes: $____  [ ] No
Stock status: [ ] Shown  [ ] Hidden
Low stock alert: [ ] Yes  [ ] No
Availability date: [ ] Shown  [ ] Hidden  [ ] N/A
```

**Expert evaluates:**

**Checklist:**
- [ ] Price prominently displayed (large font, near title)
- [ ] Compare-at/original price shown if discounted (strikethrough)
- [ ] Savings amount or percentage visible
- [ ] Stock status visible ("In Stock", "Only 3 left")
- [ ] Low stock triggers urgency (if REAL - never fake)
- [ ] "Sold Out" with restock notification option
- [ ] Per-unit price for multi-packs
- [ ] Installment option shown for high-price items
- [ ] Price includes/excludes VAT clearly stated

**Challenge questions:**
1. "Je cena jasne viditelna a velka? Nebo skryta v designu?"
2. "Ukazujes usporu pri sleve? Procento nebo castka?"
3. "Mas stock status? 'Pouze 3 skladem' zvysuje urgenci o 20% (ale MUSI byt pravdive)"
4. "Co se stane kdyz je produkt vyprodany? Nabizis notifikaci?"

**Common mistakes:**
- Price too small or poorly positioned
- No compare-at price (user doesn't see the deal)
- Fake urgency ("Only 2 left!" always) = trust destruction
- No stock status (user unsure if available)
- Sold out with no recovery (no restock alert = lost customer)
- Missing per-unit price for bundles
- No installment option for $100+ items

**Data to cite:**
- Compare-at price: +20% conversion (principle of anchoring - Kahneman)
- Real scarcity indication: +20-25% urgency (Cialdini)
- Fake urgency detected: -40% trust (Baymard)
- Restock notification: 15-30% conversion when restocked (Shopify data)
- Installment option (BNPL): +20-30% conversion for items >$100 (Klarna)

**If no stock status:**
"Chybi stock status. Uzivatel nevi jestli je produkt vubec dostupny.

**Add:**
1. 'Skladem' badge (zelena) - zakladni jistota
2. 'Pouze X skladem' kdyz X < 10 (REALNA urgence, ne fake)
3. Vyprodany -> 'Dejte mi vedet az bude' (email capture)
4. Pre-order moznost pro coming soon

**VAROVANI:** NIKDY nepouzivej fake urgency. Baymard: -40% trust kdyz uzivatel prokoukne."

**Expected impact of fixing:** +20-30% conversion

### Section 4: Buy Box

**User provides:**
```
CTA text: "_________________"
CTA position: [ ] Above fold  [ ] Below fold
Sticky on scroll: [ ] Yes  [ ] No
Variants selector: [ ] Dropdown  [ ] Visual swatches  [ ] Buttons  [ ] N/A
Quantity selector: [ ] Yes  [ ] No
```

**Expert evaluates:**

**Checklist:**
- [ ] CTA button prominent (high contrast, large)
- [ ] CTA above fold on desktop
- [ ] Sticky buy bar on scroll (keeps CTA visible)
- [ ] Variant selection via visual swatches (color, size)
- [ ] Selected variant clearly indicated
- [ ] Quantity selector present (default 1)
- [ ] "Add to Cart" not "Buy Now" (lower commitment)
- [ ] Secondary action: "Add to Wishlist" / "Save for Later"
- [ ] CTA color contrasts with page (stands out)
- [ ] Min 44x44px touch target on mobile

**Challenge questions:**
1. "Je CTA viditelny BEZ scrollovani? Sticky buy bar?"
2. "Pouzivas vizualni swatche pro varianty (barva, velikost)? Dropdowny skryvaji moznosti"
3. "Co se stane kdyz uzivatel scrollne dolu? CTA zmizi?"
4. "Je CTA 'Pridat do kosiku' nebo 'Koupit'? Nizsi commitment = vyssi konverze"

**Common mistakes:**
- CTA below fold (user must scroll to buy)
- No sticky buy bar (CTA disappears on scroll)
- Dropdown for color/size (hides options)
- "Buy Now" instead of "Add to Cart" (too aggressive)
- CTA blends with page design (low contrast)
- No wishlist option (all-or-nothing = some choose nothing)
- Variant selection unclear (what did I select?)

**Data to cite:**
- Floating/sticky buy box: +8-12% add-to-cart (Amazon, Etsy tests - GoodUI leaks)
- Visual swatches vs dropdown: +15% variant selection (Baymard)
- "Add to Cart" vs "Buy Now": +7% for Add to Cart (lower commitment - ConversionXL)
- CTA above fold: +20% visibility (NNGroup)
- High contrast CTA: +21% clicks (VWO)

**If no sticky buy bar:**
"CTA zmizi pri scrollovani. GoodUI leaks ukazuji ze Amazon i Etsy testuji floating buy box.

**Fix:**
1. Sticky buy bar (spodek stranky na mobile, sidebar na desktop)
2. Obsahuje: cena + varianta + CTA button
3. Objevi se po scrollu za buy box
4. Minimalisticky design (neodhani pozornost)

**Expected:** +8-12% add-to-cart rate. Amazon to testuje - to rika dost."

**Expected impact of fixing:** +8-21% add-to-cart

### Section 5: Reviews & Ratings

**User provides:**
```
Reviews present: [ ] Yes  [ ] No
Review count: ____
Average rating: ____

Photo reviews: [ ] Yes  [ ] No
Review filtering: [ ] By rating  [ ] By topic  [ ] None

```

**Expert evaluates:**

**Checklist:**
- [ ] Star rating visible near product title (above fold)
- [ ] Review count shown ("4.7 from 234 reviews")
- [ ] Reviews on same page (not separate page)
- [ ] Photo/video reviews highlighted
- [ ] Filter by rating (show 5-star, 3-star, 1-star)
- [ ] Sort options (newest, most helpful, rating)
- [ ] Negative reviews answered by seller
- [ ] Review highlights/summary (common pros/cons)
- [ ] "Write a Review" CTA visible
- [ ] Review distribution chart (bars showing 5/4/3/2/1 star counts)

**Challenge questions:**
1. "Jsou recenze NA strance nebo skryte v dalsi zalozce?"
2. "Reagujete na negativni recenze? To zvysuje duveru o 45%"
3. "Maji zakaznici moznost pridat fotky k recenzim?"
4. "Kde je hvezdicke hodnoceni? Musi byt u titulku (nad foldem)"

**Common mistakes:**
- Reviews on separate page (user never clicks)
- No star rating above fold (user scrolls past)
- No photo reviews (text-only = less trust)
- Negative reviews ignored (no seller response)
- No review filtering (user can't find relevant reviews)
- Only showing 5-star reviews (looks fake)
- Missing review summary/highlights
- No review distribution chart

**Data to cite:**
- Products with reviews: +270% conversion vs no reviews (Spiegel Research)
- Star rating near title: +17% add-to-cart (Shopify)
- Photo reviews: +30% purchase confidence (Bazaarvoice)
- Responding to negative reviews: +45% trust (Harvard Business Review)
- Review filtering: +22% review engagement (PowerReviews)
- Optimal rating: 4.2-4.5 stars (perfect 5.0 = suspicious - Northwestern study)

**If no reviews:**
"Produkty s recenzemi maji +270% konverzi (Spiegel Research).

**Priority:**
1. Ziskat prvnich 10-20 recenzi (follow-up email po nakupu)
2. Star rating u titulku (nad foldem)
3. Review section na strance (ne oddelena stranka)
4. Incentivizovat foto recenze (sleva na dalsi nakup)

**Pokud uz mas recenze ale spatne zobrazene:**
1. Star rating NAHORU k titulku
2. Foto recenze na prvni misto
3. Response na vsechny negativni
4. Filter + sort moznosti

**Expected:** +17-270% conversion uplift."

**Expected impact of fixing:** +17-45% add-to-cart (with existing reviews)

**User provides:**

**Expert evaluates:**

**Checklist:**
- [ ] "Customers also bought" section present
- [ ] "Frequently bought together" with one-click add-all
- [ ] Bundle deals with savings shown
- [ ] Accessories/complementary products suggested
- [ ] Cross-sell is RELEVANT (not random)
- [ ] Max 3-4 suggestions (not overwhelming)
- [ ] Cross-sell in cart as well (pre-checkout)
- [ ] Bundle savings clearly quantified ("Save $15")
- [ ] Upsell shows clear value difference

**Challenge questions:**
1. "Nabizis doplnkove produkty? 'Zakaznici take koupili' zvysuje AOV o 10-15%"
2. "Mas 'Casto kupovano spolecne' s moznosti jednoho klikku?"

4. "Nabizis bundly se slevou? Amazon generuje 35% revenue z cross-sell"

**Common mistakes:**
- No cross-sell at all (missing AOV increase)
- Random/irrelevant suggestions (hurts trust)
- Too many suggestions (10+ = overwhelming)
- No bundle option (user must add items individually)
- Cross-sell only on product page (missing cart cross-sell)
- No savings shown for bundles
- Upsell too aggressive (overshadows current product)

**Data to cite:**
- Amazon: 35% of revenue from cross-sell/recommendations
- "Frequently bought together": +10-15% AOV (Shopify data)
- Bundle deals: +20-25% AOV when savings shown (Baymard)
- Relevant cross-sell: +12% vs random recommendations (Barilliance)
- Cart cross-sell: +5% AOV (Key4You.cz Czech case study)
- Max 3-4 suggestions optimal (more = decision fatigue)

**If no cross-sell:**
"Chybi cross-sell. Amazon generuje 35% revenue z doporuceni.

**Add (priority):**
1. 'Casto kupovano spolecne' - 2-3 doplnkove produkty + 'Pridat vse' button
2. 'Zakaznici take koupili' - 3-4 relevantni produkty pod popisem
3. Bundle deal - 'Kupte 3, usetrete 15%' s jasnou usporou
4. Cart cross-sell - doplnky jeste pred checkoutem

**Pravidlo:** Relevance > pocet. 3 relevantni navrhy > 10 nahodnych.

**Expected:** +10-25% AOV increase."

**Expected impact of fixing:** +10-25% AOV

### Section 8: Mobile Product Page

**User provides:**
```
Mobile traffic share: ____%
Mobile-specific features: [ ] Gallery swipe  [ ] Sticky buy bar  [ ] Accordion details  [ ] Thumb-friendly

```

**Expert evaluates:**

**Checklist:**
- [ ] Gallery: horizontal swipe with dot indicators
- [ ] Sticky buy bar at bottom (always visible)
- [ ] Product info in accordion sections (not long scroll)
- [ ] Touch targets min 44x44px
- [ ] Font min 16px (prevents iOS zoom)
- [ ] Variant swatches large enough to tap accurately
- [ ] Price + CTA visible without scrolling
- [ ] Reviews accessible via tab (not long scroll past)

**Challenge questions:**

- Font <16px (triggers iOS auto-zoom)
- Reviews require long scroll
- No pinch-to-zoom on images

- Mobile e-commerce: 60%+ traffic, but 40-60% lower conversion than desktop
- Sticky buy bar on mobile: +15% add-to-cart (Shopify mobile checkout study)
- Accordion vs long scroll: +20% information discovery (Baymard mobile)
- Swipe gallery: +25% photo views vs thumbnails (UXPin mobile)
- Pinch zoom: 56% of users attempt it (Baymard)

**If mobile not optimized:**
"Mobil = 60%+ trafficu ale konverze je o 40-60% nizsi nez desktop.

**Priority fixes:**
1. **Sticky buy bar** - cena + CTA vzdy viditelne (bottom bar) -> +15%
2. **Swipe gallery** - horizontalni swipe s dot indicators -> +25% foto views
3. **Accordion sections** - Popis | Specs | Recenze ve sklapetkach -> +20% discovery
4. **Pinch zoom** - priblizeni fotek gestem -> povinne pro 56% uzivatelu

**Rule:** Mobilni product page NENI zmenseny desktop. Je to separatni UX.

**Expected:** +15-25% mobile add-to-cart rate."

**Expected impact of fixing:** +15-25% mobile add-to-cart
